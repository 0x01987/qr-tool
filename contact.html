What to Include in a Support Message

Detailed messages help much more than short ones. If a tool was broken, slow, or gave the wrong result, include enough information to make the issue easy to understand and reproduce.

For bug reports

Include the page URL, device type, browser name, what you entered, what you expected, and what actually happened. Screenshots are helpful when available.

For feature requests

Include the type of tool you want, who it helps, what problem it solves, and any examples of how you would use it in real life.

For content feedback

If a guide, tool description, or page explanation is unclear, mention the page and the exact wording that could be improved.

For partnerships

Include who you are, what kind of collaboration you have in mind, and how it fits with InstantQR’s tools, guides, or audience.

  • Share ideas for guides, explainers, or category hubs

Before You Send a Support Request

A few quick checks can solve common issues immediately:

  • Refresh the page and try again once
  • Try the same page in another browser
  • Check whether the page URL is correct
  • Test on desktop if the issue happened on mobile, or vice versa
  • Look for related guidance in the Guides section

If the issue still happens, sending a detailed report is the best next step.
