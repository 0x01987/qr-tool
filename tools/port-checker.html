How this tool works

The tool tries to reach a web resource on the target host and port using your browser. If the browser receives a response signal, the target is treated as reachable.

Why some results can vary

Some sites block browser requests, redirect heavily, or require authentication. That can affect the result even when the host is online.

FAQ

Can browsers scan any TCP port?

No. Browsers cannot perform raw TCP scans. This tool checks whether a browser can reach a web service on a specific port.

What ports work best?

Ports used by web services such as 80, 443, 8080, and 8443 usually work best here.

Why might a port appear unreachable?

The server may block browser requests, require authentication, redirect unusually, or use a non-browser protocol on that port.

What does reachable mean in this checker?

It means the browser got some kind of response signal from the target host and port over HTTP or HTTPS.
